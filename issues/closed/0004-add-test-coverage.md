# テストカバレッジを拡充する

Created: 2026-03-22
Completed: 2026-03-22
Model: Opus 4.6

## 概要

正式リリースに向けてテストの網羅性を大幅に向上させる。

## 背景

現在のテストは 48000Hz ステレオの無音データのみを対象としている。
unsafe コードを含むバインディングとして、より広範な入力パターンでの
正常動作を保証する必要がある。

## 対応内容

### サンプルレート網羅テスト

- 8000, 12000, 16000, 24000, 48000 の全サンプルレートでエンコード/デコードする

### チャンネル数テスト

- モノラル (channels=1) でのエンコード/デコードテスト

### フレーム長テスト

- 全 FrameDuration (2.5ms, 5ms, 10ms, 20ms, 40ms, 60ms) でのエンコード/デコードする

### Application モードテスト

- Voip, Audio, LowDelay の各モードで実際のエンコード/デコードする

### 実データテスト

- サイン波 PCM データでのラウンドトリップテスト (デコード結果が無音でないことの確認)

### 連続フレームテスト

- 100 フレーム連続エンコード/デコードの整合性テスト

### 境界値テスト

- 最小ビットレート (500) / 最大ビットレート (512000) でのエンコード
- complexity 0 と 10 でのエンコード

### エラーハンドリングテスト

- 破損パケット (ランダムバイト列) のデコードがパニックしないことの確認
- 切り詰めパケットのデコードテスト
- 空パケット (0 バイト) のデコードテスト

### スレッド安全性テスト

- Encoder / Decoder が Send を実装していることのコンパイル時確認

## 解決方法

テストを 24 個から 41 個に拡充した。全テストがサイン波 (440Hz) を使った実データ検証を行い、デコード結果を RMS で検証する。重複テスト 11 個を削除して整理した。

追加したテスト:
- サンプルレート網羅: 8000, 12000, 16000, 24000Hz
- モノラル: 48000Hz/1ch
- フレーム時間網羅: 2.5ms, 5ms, 10ms, 40ms, 60ms
- Application モード: Voip, LowDelay (Audio はデフォルトで全テストがカバー)
- 連続フレーム: 100 フレーム連続エンコード/デコード
- 境界値: 最小/最大ビットレート、complexity 0
- 複数回リセット: エンコーダー/デコーダーそれぞれ 5 回リセット
- エラーハンドリング: 破損パケット、切り詰めパケット、1 バイトパケット
- Send トレイト: コンパイル時確認
- get_lookahead の動作確認
